# Unsupervised Cluster Training Design

이 문서는 본 프로젝트의 비지도학습 파이프라인 중 **군집화 단계**를 고정하기 위한 설계 문서다.

이 문서의 범위는 아래로 제한한다.

- 표현학습 단계에서 생성된 latent vector를 로드한다
- pretrained autoencoder를 기반으로 clustering layer를 초기화한다
- IDEC 방식으로 joint training을 수행한다
- 각 beat에 최종 cluster label을 부여하여 저장한다

이 문서는 clustering 구조를 명확하게 고정하기 위한 문서이며,
이후 interpretation과 report 생성은 이 문서 범위에 포함하지 않는다.

---

# 1. 군집화 목표

군집화 단계의 목표는 **latent space 안에서 유사한 beat들이 더 조밀하게 모이도록 학습하고,
각 beat에 안정적인 cluster label을 부여하는 것**이다.

이 단계의 핵심은 단순히 latent vector에 외부 clustering만 수행하는 것이 아니라,
**reconstruction loss와 clustering loss를 동시에 사용하여 latent space 자체를 cluster-friendly하게 재정렬하는 것**이다.

즉, 이 단계는 IDEC 방식의 joint training으로 고정한다.

---

# 2. 입력 파일 정의

군집화 단계의 입력은 아래 파일로 고정한다.

- `beat_features_valid.csv`
- `learning_input_columns.json`
- `latent_valid.csv`
- `dae_best.pt`
- `scaler.joblib`
- `split_info.json`

모델 입력은 전처리 단계에서 확정된 valid beat feature이며,
representation learning 단계에서 저장된 scaler를 동일하게 사용한다.

---

# 3. clustering 방식 고정

본 프로젝트의 군집화 방식은 **IDEC (Improved Deep Embedded Clustering)** 로 고정한다.

즉, 아래 구조를 사용한다.

1. pretrained denoising autoencoder 로드
2. encoder output을 latent representation으로 사용
3. latent space 위에 cluster center를 둔다
4. reconstruction loss와 clustering loss를 동시에 최적화한다

이 문서에서는 다른 clustering 구조를 사용하지 않는다.

---

# 4. cluster 개수 고정

cluster 개수는 아래로 고정한다.

- `NUM_CLUSTERS = 4`

즉, 모든 valid beat는 최종적으로 아래 중 하나의 label을 가진다.

- `cluster_0`
- `cluster_1`
- `cluster_2`
- `cluster_3`

cluster label은 정수 0~3으로 저장한다.

---

# 5. 초기 cluster center 설정

초기 cluster center는 pretrained autoencoder의 latent space에서 설정한다.

초기화 절차는 아래로 고정한다.

1. `dae_best.pt` 로드
2. train split valid beat를 encoder에 통과시켜 latent vector 생성
3. train latent vector에 대해 `KMeans(n_clusters=4)` 수행
4. KMeans 중심을 clustering layer 초기 center로 설정

초기 중심점 계산은 train split에 대해서만 수행한다.

---

# 6. IDEC soft assignment 정의

각 latent vector `z_i`와 cluster center `μ_j`에 대해
DEC 방식의 Student-t soft assignment를 사용한다.

soft assignment `q_ij`는 아래 규칙으로 계산한다.

- 각 beat가 각 cluster 중심에 속할 확률 형태의 soft value를 가진다
- 모든 cluster에 대한 `q_ij` 합은 1이 되도록 정규화한다

target distribution `p_ij`는 DEC 방식으로 계산한다.

- 높은 confidence assignment를 더 강화하는 방향으로 target distribution을 만든다

구현은 DEC / IDEC 표준식으로 고정한다.

---

# 7. loss 정의 고정

군집화 단계의 total loss는 아래로 고정한다.

- `L_total = L_recon + beta * L_cluster`

각 항목은 아래를 의미한다.

## 7.1 reconstruction loss
- `L_recon = MSELoss(decoded_x, clean_scaled_x)`

즉, autoencoder가 입력 feature 구조를 계속 유지하도록 한다.

## 7.2 clustering loss
- `L_cluster = KL(P || Q)`

즉, target distribution `P`와 soft assignment `Q` 간 KL divergence를 최소화한다.

## 7.3 beta 고정
- `beta = 0.1`

이 값은 reconstruction 구조를 유지하면서도 clustering 효과를 주기 위한 고정값으로 사용한다.

---

# 8. 모델 구조 고정

군집화 단계의 backbone은 표현학습 단계에서 사용한 denoising autoencoder와 동일하다.

즉,

- encoder
- decoder
- latent dimension 12

를 그대로 유지한다.

여기에 아래 요소를 추가한다.

- learnable clustering centers: shape `(4, 12)`

이 중심점은 joint training 동안 함께 업데이트된다.

---

# 9. 학습 데이터 사용 규칙

IDEC joint training은 train split을 기준으로 수행한다.

validation split은 model selection 용도로만 사용한다.

test split은 joint training에 사용하지 않는다.

즉, 사용 방식은 아래로 고정한다.

- train: 파라미터 업데이트
- validation: early stopping 및 best model 선택
- test: 학습 완료 후 평가용
- 전체 valid beat: 최종 label export용

---

# 10. 입력 생성 규칙

군집화 단계의 입력은 scaling된 clean feature를 사용한다.

denoising 입력은 이 단계에서 다시 사용하지 않는다.

즉, joint training 단계에서는 아래를 사용한다.

- 입력: clean scaled feature
- reconstruction target: clean scaled feature

표현학습 단계에서 만든 robust scaling 결과를 그대로 사용한다.

---

- batch size: `256`
- max epochs: `100`
- early stopping patience: `15`
- optimizer: `Adam`
- learning rate: `1e-4`
- weight decay: `1e-5`

best model 선택 기준은 아래로 고정한다.

- validation total loss 최소

---

# 12. target distribution 갱신 규칙

target distribution은 매 epoch마다 전체 train latent를 기준으로 갱신한다.

즉, epoch 시작 시 아래 절차를 수행한다.

1. train feature를 encoder에 통과시켜 latent 생성
2. current cluster center로 soft assignment `Q` 계산
3. `Q`로부터 target distribution `P` 계산
4. 해당 epoch 학습에 `P` 사용

epoch 중간 minibatch마다 전체 `P`를 다시 계산하지 않는다.
epoch 단위 갱신으로 고정한다.

---

# 13. 최종 cluster label 생성 규칙

최종 cluster label은 best IDEC checkpoint 기준으로 생성한다.

전체 valid beat에 대해 아래 절차를 수행한다.

1. scaled valid feature를 encoder에 통과시켜 latent 생성
2. cluster center에 대한 soft assignment `Q` 계산
3. `argmax(Q)`를 final cluster label로 저장

각 beat에 대해 아래 값들을 함께 저장한다.

- `cluster_label`
- `cluster_confidence`
- `q_cluster_0`
- `q_cluster_1`
- `q_cluster_2`
- `q_cluster_3`

`cluster_confidence`는 `max(Q)`로 정의한다.

# 18. 군집화 완료 상태 정의

군집화가 완료되면 아래가 만족되어야 한다.

- pretrained DAE를 기반으로 IDEC joint training이 수행된다
- cluster center가 train latent에서 초기화된다
- reconstruction loss와 clustering loss가 함께 최적화된다
- best IDEC checkpoint가 저장된다
- 전체 valid beat에 final cluster label이 부여된다
- cluster confidence 및 soft assignment가 함께 저장된다

이 상태를 군집화 완료 상태로 정의한다.
