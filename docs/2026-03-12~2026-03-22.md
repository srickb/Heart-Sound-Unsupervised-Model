# 2026-03-12~2026-03-22 연구 진행 정리

## 개요

이 기간의 현재 연구 진행 방향은 raw waveform 자체를 바로 비지도학습하는 것이 아니라,
심장음 recording을 cycle 단위로 분할한 뒤 각 cycle을 설명 가능한 수치형 feature로 변환하고,
그 feature 공간에서 autoencoder와 HDBSCAN을 이용해 구조를 찾는 방향으로 정리되어 있습니다.

즉, 핵심 목표는 심장음 cycle의 구조적 차이를 해석 가능한 형태로 표현하고,
비지도학습을 통해 유사한 cycle pattern을 군집화하며,
이후 cluster의 의미를 feature와 waveform 관점에서 해석 가능한 결과로 정리하는 것입니다.

## 현재 파이프라인 구조

현재 프로젝트의 메인 파이프라인은 아래 순서로 구성되어 있습니다.

1. `01_preprocess.py`
2. `02_train_autoencoder.py`
3. `03_extract_embeddings_and_hdbscan.py`
4. `04_interpret_clusters.py`
5. `05_export_excel_report.py`
6. `06_export_cycle_cluster_assignments.py`

이 구조는 전처리, 학습, 임베딩 추출 및 군집화, 해석, Excel 보고서 정리까지 하나의 흐름으로 이어지는 standalone 파이프라인 형태입니다.

## 단계별 진행 내용

### 1. 전처리 단계

전처리 단계의 핵심은 단순 segmentation이 아니라,
각 심장 cycle을 비지도학습에 적합한 feature vector로 변환하는 것입니다.

현재 코드에서는 다음 흐름으로 진행됩니다.

- 입력은 Excel 기반 recording 파일입니다.
- `S1-Start_RS_Score`, `S1-End_RS_Score`, `S2-Start_RS_Score`, `S2-End_RS_Score`를 이용해 cycle 내부 event를 찾습니다.
- 기본 cycle 정의는 `S1(i) 시작 ~ S1(i+1) 시작 전`입니다.
- 각 cycle에 대해 valid 여부, exclusion reason, event candidate 개수, timing 정보 등을 metadata로 저장합니다.

현재 feature engineering 방향은 다음 여섯 그룹으로 정리됩니다.

- `rs_`: landmark 시점 RS score 및 local window 통계
- `time_`: cycle duration, S1/S2/systole/diastole 구간 시간 정보
- `amp_raw_`: raw amplitude 기반 통계
- `amp_norm_`: normalized amplitude 기반 통계
- `ratio_`: 시간/진폭 비교 비율 feature
- `qc_`: candidate 개수, ordering flag, cycle length bounds 등 품질 관련 feature

즉, 이 프로젝트의 전처리 단계는 raw waveform을 그대로 모델에 넣기 위한 준비가 아니라,
cycle 구조를 설명 가능한 숫자 표현으로 바꾸는 feature engineering 단계라고 볼 수 있습니다.

### 2. Autoencoder 학습 단계

학습 단계는 raw waveform reconstruction 모델이 아니라,
cycle-level tabular feature를 입력으로 받는 dense autoencoder를 학습하는 구조입니다.

현재 학습 단계의 특징은 다음과 같습니다.

- 입력은 `cycle_features.npy`입니다.
- valid cycle만 학습 대상으로 사용합니다.
- train/validation split은 가능하면 `subject_id` 기준,
  그렇지 않으면 `recording_id` 기준으로 나눕니다.
- normalization은 train split에 대해서만 적응하도록 설계되어 있습니다.
- hidden layer는 작은 dense 구조이며 latent dimension은 현재 8입니다.

즉, 이 단계의 목적은 feature를 더 작은 잠재 표현으로 압축하면서도,
cycle 간 구조 차이를 유지하는 latent space를 만드는 것입니다.

### 3. Embedding 추출 및 HDBSCAN 단계

세 번째 단계에서는 학습된 encoder를 이용해 각 cycle의 latent embedding을 추출하고,
그 잠재 공간에서 HDBSCAN으로 군집을 찾습니다.

현재 방향의 핵심은 다음과 같습니다.

- cluster 개수를 직접 정하지 않습니다.
- density 기반으로 안정적인 group을 찾습니다.
- 애매한 sample은 `noise`로 남길 수 있도록 설계되어 있습니다.
- `membership_probability`, `outlier_score` 같은 보조 정보도 함께 저장합니다.

즉, 이 단계는 정상/비정상 이분법보다,
cycle pattern이 자연스럽게 어떤 군집 구조를 이루는지 보는 단계입니다.

### 4. Cluster 해석 단계

현재 연구 방향에서 가장 중요한 부분 중 하나는 interpretation 단계입니다.

이 단계에서는 다음을 수행합니다.

- cluster별 sample 수와 비율 정리
- feature group별 주요 차이 분석
- 대표 sample 추출
- noise group의 artifact-like 성격 추정
- waveform panel과 boxplot 생성
- markdown/Excel 기반 해석 결과 저장

중요한 점은,
현재 코드는 cluster를 자동으로 `normal`, `murmur` 같은 임상 라벨로 단정하지 않고,
중립적인 이름인 `cluster_0`, `cluster_1`, `noise` 형태로 유지하도록 설계되어 있다는 점입니다.

즉, 현재 해석 단계의 목적은 진단 분류가 아니라,
비지도학습이 찾아낸 cycle 구조를 feature 차이와 waveform 경향으로 설명하는 것입니다.

### 5. 결과 정리 및 Excel export 단계

후반 단계에서는 사람이 바로 볼 수 있도록 Excel 형태의 결과물을 정리합니다.

현재 구성은 다음 두 방향으로 나뉩니다.

- 전체 파이프라인 결과를 한 workbook으로 요약하는 보고서
- 각 recording별 cycle이 어떤 cluster 또는 noise로 분류되었는지를 정리한 workbook

즉, 현재 프로젝트는 단순 모델 코드가 아니라,
연구자가 실험 결과를 직접 검토하고 해석할 수 있는 분석 파이프라인으로 확장되고 있습니다.

## 현재 연구 방향의 핵심 해석

현재 코드와 아이디어 문서를 종합하면,
이 프로젝트의 진짜 목표는 아래와 같이 정리됩니다.

- 심장음 cycle을 구조적으로 설명 가능한 feature 공간으로 옮기는 것
- feature 공간에서 저차원 latent representation을 학습하는 것
- latent space에서 HDBSCAN으로 안정적인 cycle pattern group을 찾는 것
- 각 cluster가 시간 구조, RS landmark, amplitude, QC 관점에서 어떤 의미를 가지는지 해석하는 것

즉, 최종 산출물은 분류 정확도 하나가 아니라,
`cycle archetype`, `대표 sample`, `군집 간 feature 차이`, `noise/artifact 후보`를 연구적으로 해석 가능한 형태로 정리하는 것에 가깝습니다.

## 현재 구조의 강점

현재 연구 구조의 강점은 다음과 같습니다.

- feature 기반이라 해석 가능성이 높습니다.
- raw amplitude와 normalized amplitude를 분리해 shape와 크기 정보를 함께 봅니다.
- HDBSCAN을 사용해 cluster 수를 사전에 고정하지 않아도 됩니다.
- noise를 별도 그룹으로 다루기 때문에 애매한 cycle을 억지로 cluster에 넣지 않습니다.
- interpretation과 Excel export까지 포함되어 있어 결과 검토 흐름이 좋습니다.

## 현재 구조의 한계 및 주의점

현재 코드 기준으로 보이는 한계도 있습니다.

- missing event, invalid ordering, empty segment는 여전히 제외되는 경우가 많아,
  이상 cycle 자체를 연구 대상으로 삼고 싶을 때는 exclusion 정책을 더 완화할 여지가 있습니다.
- 모델 입력이 끝까지 tabular feature 기반이므로,
  raw waveform의 미세한 morphology 정보는 일부 손실될 수 있습니다.
- 일부 config 값은 선언되어 있지만 실제 분기에서 완전히 활용되지는 않는 흔적이 있습니다.
- interpretation은 잘 설계되어 있지만,
  cluster 의미는 여전히 descriptive level이며 임상적 결론으로 바로 연결되지는 않습니다.

## 현재 워크스페이스 기준 상태

현재 이 저장소를 확인한 시점 기준으로는 `Data` 폴더와 `outputs` 폴더가 워크스페이스에 존재하지 않았습니다.
따라서 이번 정리는 실제 실행 결과 수치보다는,
코드 구조와 `Idea_DataBase` 문서에 적힌 연구 의도를 바탕으로 정리한 진행 상황 분석입니다.

즉, 현재 확인 가능한 것은 다음 두 가지입니다.

- 연구 방향과 코드 설계는 비교적 일관되게 정리되어 있습니다.
- 실제 실험 결과물은 현재 워크스페이스에는 남아 있지 않아,
  cluster 개수나 noise 비율 같은 최신 수치는 이 문서에 확정적으로 적지 않았습니다.

## 종합 정리

2026-03-12부터 2026-03-22까지의 현재 연구 진행 방향은,
심장음 cycle을 해석 가능한 feature 기반 표현으로 바꾸고,
그 위에 autoencoder와 HDBSCAN을 올려 비지도 구조를 찾은 뒤,
cluster의 의미를 feature 및 waveform 관점에서 해석 가능한 연구 파이프라인으로 정리하는 과정이라고 볼 수 있습니다.

즉, 이 프로젝트는 단순한 비지도 clustering 코드가 아니라,
심장음 cycle pattern discovery와 해석 가능성을 함께 추구하는 연구형 분석 시스템으로 발전하고 있습니다.
