# Unsupervised Cluster Interpretation Design

이 문서는 본 프로젝트의 비지도학습 파이프라인 중 **cluster interpretation 및 report 생성 단계**를 고정하기 위한 설계 문서다.

이 문서의 범위는 아래로 제한한다.

- cluster assignment 결과를 로드한다
- cluster별 feature distribution을 요약한다
- cluster별 대표 beat를 선택한다
- feature group별 차이를 정리한다
- 최종 해석용 report와 export 파일을 생성한다

이 문서는 cluster interpretation 구조를 명확하게 고정하기 위한 문서이며,
추가 학습이나 재학습은 이 문서 범위에 포함하지 않는다.

---

# 1. interpretation 목표

interpretation 단계의 목표는 **각 cluster가 어떤 beat 특성을 기준으로 형성되었는지 설명 가능한 형태로 정리하는 것**이다.

즉, 이 단계는 아래 질문에 답할 수 있어야 한다.

- cluster마다 beat 수와 비율은 어떤가
- cluster마다 timing 구조가 어떻게 다른가
- cluster마다 amplitude / shape / complexity / stability가 어떻게 다른가
- 각 cluster를 대표하는 beat는 무엇인가
- 어떤 feature가 cluster 분리에 크게 기여했는가

이 단계의 출력은 사람이 읽고 이해할 수 있는 summary table과 report다.

---

# 2. 입력 파일 정의

interpretation 단계의 입력은 아래 파일로 고정한다.

- `beat_features_valid.csv`
- `learning_input_columns.json`
- `cluster_assignments_valid.csv`
- `feature_names.json`
- `cluster_centers.npy`

필요 시 아래 파일도 함께 사용한다.

- `beat_features_all.csv`
- `record_summary.csv`

---

# 3. 병합 규칙

해석 단계의 기본 table은 아래 두 파일을 key 기준으로 merge하여 생성한다.

- `beat_features_valid.csv`
- `cluster_assignments_valid.csv`

merge key는 아래로 고정한다.

- `record_id`
- `beat_index`

merge 후 생성되는 master table을 아래로 정의한다.

- `clustered_valid_beats`

이 table은 interpretation 단계의 기준 table이다.

---

# 4. cluster label 규칙

cluster label은 아래 형식으로 유지한다.

- `0`
- `1`
- `2`
- `3`

출력 report에서는 아래 이름을 함께 사용한다.

- `cluster_0`
- `cluster_1`
- `cluster_2`
- `cluster_3`

자동으로 의미 이름을 붙이지 않는다.

---

# 5. cluster 기본 통계 생성 규칙

각 cluster에 대해 아래 항목을 반드시 계산한다.

- beat count
- 전체 valid beat 대비 비율
- 평균 cluster confidence
- confidence standard deviation

출력 파일명 예시:
- `cluster_overview.csv`

포함 column 예시:
- `cluster_label`
- `beat_count`
- `beat_ratio`
- `confidence_mean`
- `confidence_std`

---

# 6. feature summary 규칙

각 cluster에 대해 모든 학습 입력 feature에 대해 아래 summary를 계산한다.

- mean
- std
- median
- q25
- q75

global 기준 summary도 함께 계산한다.

즉, 각 feature에 대해 아래 두 수준의 summary가 존재해야 한다.

1. global summary
2. cluster-wise summary

출력 파일명 예시:
- `feature_summary_by_cluster.csv`

---

# 7. cluster separation feature 규칙

cluster 해석을 위해 각 cluster별 **대표 차이 feature**를 반드시 계산한다.

각 cluster와 global distribution을 비교하여 아래 값을 계산한다.

- `delta_mean = cluster_mean - global_mean`
- `effect_z = (cluster_mean - global_mean) / (global_std + eps)`

각 cluster마다 `abs(effect_z)`가 큰 상위 10개 feature를 추출한다.

출력 파일명 예시:
- `top_features_per_cluster.csv`

포함 항목:
- `cluster_label`
- `feature_name`
- `cluster_mean`
- `global_mean`
- `delta_mean`
- `effect_z`
- `rank_within_cluster`

이 표는 cluster가 어떤 feature 방향으로 분리되었는지 설명하는 핵심 표로 사용한다.

---

# 8. feature group summary 규칙

feature prefix 기준으로 group summary를 만든다.

group은 아래로 고정한다.

- `time_`
- `amp_`
- `shape_`
- `stat_`
- `stab_`

각 cluster에 대해 group별로 아래 값을 계산한다.

- group feature 수
- group 내 feature들의 absolute effect_z 평균
- group 내 상위 5개 대표 feature 목록

출력 파일명 예시:
- `feature_group_summary_by_cluster.json`

이 summary는 각 cluster가 주로 어떤 group 차이로 형성되었는지 빠르게 보여주기 위한 것이다.

---

# 9. 대표 beat 선택 규칙

각 cluster의 대표 beat는 **latent medoid 방식**으로 선택한다.

절차는 아래로 고정한다.

1. 해당 cluster에 속한 beat들의 latent vector를 모은다
2. cluster center와의 Euclidean distance를 계산한다
3. 거리가 가장 작은 beat를 `cluster medoid beat`로 선택한다

각 cluster마다 아래를 선택한다.

- 대표 beat 1개: medoid beat
- 추가 대표 beat 5개: center에 가장 가까운 상위 5개 beat

출력 파일명 예시:
- `representative_beats.csv`

포함 항목:
- `cluster_label`
- `record_id`
- `beat_index`
- `rank_in_cluster`
- `distance_to_center`
- `cluster_confidence`

---

# 10. 대표 beat waveform export 규칙

대표 beat에 대해서는 waveform과 envelope를 시각화하여 저장한다.

각 representative beat에 대해 아래 그림을 생성한다.

- raw filtered PCG beat segment
- S1 / S2 boundary 표시
- envelope overlay

파일 저장 구조 예시:
- `figures/cluster_0/representative_rank_1.png`
- `figures/cluster_0/representative_rank_2.png`
- ...
- `figures/cluster_3/representative_rank_5.png`

시각화는 interpretation 단계의 필수 산출물이다.

---

# 11. record-level cluster distribution 규칙

각 recording이 어떤 cluster 분포를 가지는지 요약한다.

각 `record_id`에 대해 아래를 계산한다.

- total valid beat count
- cluster_0 count / ratio
- cluster_1 count / ratio
- cluster_2 count / ratio
- cluster_3 count / ratio

출력 파일명 예시:
- `record_cluster_distribution.csv`

이 표는 특정 recording이 어떤 beat subtype 구성을 가지는지 확인하기 위한 것이다.

---

# 12. 2D latent visualization 규칙

cluster 시각화를 위해 2D latent visualization을 생성한다.

사용 방법은 아래로 고정한다.

- 입력: `latent_00` ~ `latent_11`
- 차원 축소: `UMAP`
- output dimension: 2
- random seed: 42

UMAP 2D 결과에 대해 아래 scatter plot을 저장한다.

- color = cluster_label
- alpha 고정
- legend 포함

파일명 예시:
- `figures/latent_umap_by_cluster.png`

이 그림은 visualization 전용이며 clustering 계산에는 사용하지 않는다.

---

# 13. Excel report 규칙

최종 interpretation 결과는 Excel workbook으로도 저장한다.

파일명 예시:
- `cluster_interpretation_report.xlsx`

반드시 아래 시트를 포함한다.

- `cluster_overview`
- `top_features`
- `feature_summary`
- `representative_beats`
- `record_cluster_distribution`

---

# 14. JSON summary 규칙

최종 자동 요약용 JSON도 생성한다.

파일명 예시:
- `cluster_interpretation_summary.json`

각 cluster마다 아래 항목을 포함한다.

- beat count
- beat ratio
- confidence mean
- top feature 10개
- feature group summary
- representative beat 정보

이 JSON은 사람이 읽는 용도와 후속 자동 보고 용도를 동시에 가진다.

---

# 15. logging 규칙

반드시 아래 로그를 남긴다.

- input file load 완료
- merge 완료
- cluster별 beat 수
- feature summary 생성 완료
- top feature 추출 완료
- representative beat 선택 완료
- UMAP figure 저장 완료
- Excel / JSON report 저장 완료

print 대신 logging을 사용한다.

---

# 16. config 구조 고정

interpretation 코드는 파일 상단에 class 기반 config를 둔다.

필수 항목은 아래와 같다.

- `PROJECT_ROOT`
- `INPUT_ROOT`
- `OUTPUT_ROOT`
- `RUN_NAME`
- `RANDOM_SEED`
- `NUM_CLUSTERS`
- `TOP_FEATURES_PER_CLUSTER`
- `REPRESENTATIVE_BEATS_PER_CLUSTER`
- `UMAP_N_NEIGHBORS`
- `UMAP_MIN_DIST`

기본값은 아래로 고정한다.

- `NUM_CLUSTERS = 4`
- `TOP_FEATURES_PER_CLUSTER = 10`
- `REPRESENTATIVE_BEATS_PER_CLUSTER = 5`
- `UMAP_N_NEIGHBORS = 15`
- `UMAP_MIN_DIST = 0.1`

---

# 17. 구현 순서 고정

interpretation 단계는 반드시 아래 순서로 구현한다.

1. 입력 파일 로드
2. valid feature와 cluster assignment merge
3. cluster overview 계산
4. feature summary 계산
5. top feature per cluster 계산
6. feature group summary 생성
7. representative beat 선택
8. representative waveform figure 저장
9. record-level cluster distribution 생성
10. UMAP 시각화 생성
11. Excel / JSON / CSV report 저장

이 순서를 유지한다.

---

# 18. interpretation 완료 상태 정의

interpretation이 완료되면 아래가 만족되어야 한다.

- cluster별 beat 수와 비율이 정리된다
- cluster별 대표 feature 차이가 정리된다
- cluster별 대표 beat가 선택된다
- waveform figure가 저장된다
- recording별 cluster composition이 정리된다
- UMAP 기반 2D visualization이 저장된다
- Excel / JSON / CSV report가 생성된다

이 상태를 interpretation 완료 상태로 정의한다.
