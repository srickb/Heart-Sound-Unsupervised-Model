# Unsupervised Representation Learning Design

이 문서는 본 프로젝트의 비지도학습 파이프라인 중 **표현학습 단계**를 고정하기 위한 설계 문서다.

이 문서의 범위는 아래로 제한한다.

- 전처리 결과에서 valid beat feature를 로드한다
- 학습 입력 column을 확정한다
- scaling을 수행한다
- denoising autoencoder를 학습한다
- 각 beat의 latent representation을 추출하여 저장한다

이 문서는 표현학습 구조를 명확하게 고정하기 위한 문서이며,
이후 clustering과 interpretation은 이 문서 범위에 포함하지 않는다.

---

# 1. 표현학습 목표

표현학습의 목표는 **각 valid beat feature vector를 저차원 latent vector로 압축하는 것**이다.

이 latent vector는 아래 조건을 만족해야 한다.

- beat의 timing / amplitude / shape / statistics / stability 구조를 유지해야 한다
- 유사한 beat는 latent space에서 가깝게 위치해야 한다
- reconstruction이 가능해야 한다
- 이후 clustering 단계의 입력으로 직접 사용할 수 있어야 한다

즉, 이 단계의 출력은 **각 beat를 설명하는 compressed representation**이다.

---

# 2. 입력 파일 정의

표현학습 단계의 입력은 전처리 단계에서 생성된 아래 파일로 고정한다.

- `beat_features_valid.csv`
- `learning_input_columns.json`
- `feature_names.json`

필수 metadata column은 아래를 포함한다.

- `record_id`
- `beat_index`
- `valid_flag`

학습 입력 feature는 반드시 `learning_input_columns.json` 기준으로 선택한다.

직접 column 이름을 하드코딩하지 않는다.

---

# 3. 데이터 분할 규칙

데이터 분할은 **record 단위 group split**으로 고정한다.

같은 recording에서 나온 beat가 train / validation / test에 동시에 섞이지 않도록 한다.

분할 비율은 아래로 고정한다.

- train: 70%
- validation: 15%
- test: 15%

분할 기준 key는 `record_id`다.

random seed는 고정한다.

- `RANDOM_SEED = 42`

이 단계에서 사용하는 모든 split은 record 기준으로만 수행한다.

---

# 4. 학습 입력 column 규칙

학습 입력 column은 반드시 `learning_input_columns.json`에 기록된 순서 그대로 사용한다.

다음 column은 학습 입력에 포함하지 않는다.

- `record_id`
- `beat_index`
- `valid_flag`
- boundary index metadata
- absolute time metadata
- summary column

입력 차원 `D`는 아래로 정의한다.

- `D = len(learning_input_columns)`

이 값은 코드에서 자동 계산해야 한다.

---

# 5. scaling 규칙

feature 저장값은 원 단위를 유지하지만,
모델 입력은 scaling 후 사용한다.

scaling은 아래 방식으로 고정한다.

1. train split의 valid beat feature만 사용하여 scaler를 fit
2. validation / test에는 동일 scaler를 transform만 적용
3. scaler는 저장하여 이후 재사용

기본 scaler는 아래로 고정한다.

- `RobustScaler`

저장 파일명 예시는 아래와 같다.

- `scaler.joblib`

---

# 6. denoising 규칙

표현학습은 단순 autoencoder가 아니라 **denoising autoencoder**로 수행한다.

입력 feature는 scaling된 값을 사용한다.

denoising 방식은 아래로 고정한다.

- 입력 feature의 15%를 random masking 한다
- masking된 위치의 입력값은 0으로 치환한다
- 모델은 원래의 clean input을 복원하도록 학습한다

즉, 모델 입력은 noisy input이고,
loss 계산 대상은 clean scaled input이다.

---

# 7. 모델 구조 고정

표현학습 모델은 **MLP 기반 denoising autoencoder**로 고정한다.

## 7.1 encoder 구조

입력 차원은 `D`다.

encoder는 아래 구조로 고정한다.

- Linear(`D`, 128)
- ReLU
- Linear(128, 64)
- ReLU
- Linear(64, 32)
- ReLU
- Linear(32, 12)

최종 latent dimension은 아래로 고정한다.

- `LATENT_DIM = 12`

## 7.2 decoder 구조

decoder는 아래 구조로 고정한다.

- Linear(12, 32)
- ReLU
- Linear(32, 64)
- ReLU
- Linear(64, 128)
- ReLU
- Linear(128, `D`)

decoder output activation은 두지 않는다.

최종 출력은 scaled feature space에서 원 입력을 복원하는 값이다.

---

# 8. loss 및 optimizer 규칙

학습 loss는 아래로 고정한다.

- `MSELoss`

즉, decoder output과 clean scaled input 간 mean squared error를 최소화한다.

optimizer는 아래로 고정한다.

- `Adam`

하이퍼파라미터는 아래로 고정한다.

- learning rate: `1e-3`
- weight decay: `1e-5`

---

# 9. 학습 하이퍼파라미터 고정

학습 설정은 아래로 고정한다.

- batch size: `256`
- max epochs: `100`
- early stopping patience: `15`
- model selection 기준: validation reconstruction loss 최소

best validation loss를 기록한 epoch의 모델만 최종 모델로 저장한다.

---

# 10. 저장 규칙

표현학습 단계가 끝나면 아래 산출물을 저장한다.

## 10.1 model checkpoint
- `dae_best.pt`

## 10.2 scaler
- `scaler.joblib`

## 10.3 training history
- `training_history.csv`

포함 항목:
- epoch
- train_loss
- val_loss

## 10.4 latent representation
valid beat 전체에 대해 latent vector를 추출하여 저장한다.

파일명 예시:
- `latent_valid.csv`

포함 항목:
- `record_id`
- `beat_index`
- `latent_00`
- `latent_01`
- ...
- `latent_11`

## 10.5 split metadata
파일명 예시:
- `split_info.json`

포함 항목:
- train record 목록
- validation record 목록
- test record 목록
- random seed

---

# 11. latent extraction 규칙

latent extraction은 반드시 best checkpoint 기준으로 수행한다.

대상은 아래로 고정한다.

- `beat_features_valid.csv`의 전체 valid beat

즉, train/validation/test를 모두 포함한 전체 valid beat에 대해 latent를 추출한다.

latent 추출 시 encoder의 최종 출력 12차원 vector를 그대로 사용한다.

---

# 12. logging 규칙

반드시 아래 로그를 남긴다.

- 입력 파일 로드 시작/완료
- 전체 valid beat 수
- train / validation / test record 수
- train / validation / test beat 수
- input dimension
- scaler fit 완료
- epoch별 train / validation loss
- best epoch
- latent export 경로

print 대신 logging을 사용한다.

---

# 14. 구현 순서 고정

표현학습 단계는 반드시 아래 순서로 구현한다.

1. 전처리 출력 파일 로드
2. learning input column 로드
3. record 기준 split 생성
4. train 기준 scaler fit
5. train / validation / test transform
6. denoising autoencoder 구성
7. model 학습
8. best checkpoint 저장
9. 전체 valid beat에 대해 latent 추출
10. 결과 파일 저장

이 순서를 유지한다.

---

# 15. 표현학습 완료 상태 정의

표현학습이 완료되면 아래가 만족되어야 한다.

- valid beat feature가 train/validation/test로 record 기준 분리된다
- scaler가 train 기준으로 fit 되어 저장된다
- denoising autoencoder가 학습된다
- best checkpoint가 저장된다
- 전체 valid beat에 대한 12차원 latent vector가 저장된다
- training history가 저장된다

이 상태를 표현학습 완료 상태로 정의한다.
